==========================================
  DIAGNÓSTICO ERP — Gigasistemas
==========================================

PRA QUE SERVE
-------------
Inspecionar o schema de uma tabela do Gigasistemas (MySQL on-premise)
direto, SEM precisar do backend NestJS rodando. Útil pra descobrir
estrutura de tabelas antes de escrever queries definitivas (ex: auto-match
de VENDA CERTA via PRODVENDIDOS).


COMO USAR
---------
1. Essa pasta precisa estar dentro de flowops-lite/tools/diagnostico-erp/
   (no mesmo repo que tem backend/.env com as credenciais ERP_*)

2. LISTAR TODAS AS TABELAS do banco (quando não sabe o nome certo):
   diagnostico-erp.bat --list

3. BUSCAR tabelas por palavra-chave:
   diagnostico-erp.bat --find VEND
   diagnostico-erp.bat --find PROD
   diagnostico-erp.bat --find VEND,CUPOM   (vários termos separados por vírgula)

4. Pra INSPECIONAR uma tabela específica (colunas + 3 linhas de amostra):
   diagnostico-erp.bat NOME_DA_TABELA

   Exemplos:
     diagnostico-erp.bat PRODUTOS
     diagnostico-erp.bat ESTOQUE
     diagnostico-erp.bat CLIENTES

5. Duplo-clique no .bat (sem args) → tenta tabela PRODVENDIDOS por default

FLUXO RECOMENDADO quando não sabe o nome:
   1. diagnostico-erp.bat --find VEND       → vê candidatos que têm "VEND"
   2. diagnostico-erp.bat NOME_ESCOLHIDO    → inspeciona colunas + amostra


O QUE O SCRIPT FAZ
------------------
1. Lê as credenciais ERP_HOST/PORT/USER/PASSWORD/DATABASE do backend/.env
2. Conecta no MySQL do Gigasistemas
3. Roda SHOW COLUMNS FROM <tabela>
4. Roda SELECT * FROM <tabela> LIMIT 3
5. Imprime o resultado no console
6. Salva tudo em resultado.json pra facilitar copiar


REQUISITOS
----------
- Node.js instalado (https://nodejs.org)
- backend/.env preenchido com ERP_HOST, ERP_USER, ERP_PASSWORD, ERP_DATABASE
- mysql2 instalado no backend (o .bat instala automaticamente se faltar)
- Máquina onde roda precisa ter acesso de rede ao Gigasistemas


PROBLEMAS COMUNS
----------------
[ECONNREFUSED / ETIMEDOUT]
  Gigasistemas desligado ou fora da rede. Verifique ERP_HOST.

[ER_ACCESS_DENIED_ERROR]
  Usuário/senha errados no backend/.env.

[ER_NO_SUCH_TABLE]
  Nome da tabela errado. Confira no Gigasistemas.


RESULTADO
---------
Gera resultado.json na mesma pasta. Copie esse arquivo inteiro pra
compartilhar com quem precisar inspecionar o schema.
